SQLMapper Module

To map the AST JSON to the other DBMS query. DBMS mapping supported: CouchDB (PouchDB & Mango query), Neo4j(cypher query).

1. Usage

Using in Node.js

var SQLParser = require('./dbAPI-server').SQLMapper;

//CouchDB                            
SQLMapper.couchdb(SQLtype, connection, ast, function (err, result) {
    //Handle err & result
});

//Neo4j                            
SQLMapper.neo4j(SQLtype, ast, function (err, result) {
    //Handle err & result
});

2. Input

Parameters meaning:
  • SQLtype: create/select/insert/update/delete/alter/index
  • connection (currently implement for CouchDB):the connection instance is created by dbConnect module. Used for DBMS which need to find data before mapping to update/delete query (like CouchDB). This param can be null for other case. See dbConnect module for detail.
  • ast: The AST JSON, ouput from SQLParser module. See SQLParser module for detail.

3. Output

The Neo4j(Cypher query) & CouchDB(Mango query) mapping result:

  "variant": "list",
  "statement": [

          "type": "identifier",
          "variant": "star",
          "name": "*"
        }
      ],
      "from": {
        "type": "identifier",
        "variant": "table",
        "name": "users"
      },
      "where": [
        {
          "type": "expression",
          "format": "binary",
          "variant": "operation",
          "operation": "=",
          "left": {
            "type": "identifier",
            "variant": "column",
            "name": "name"

          "right": {
            "type": "identifier",

====> Neo4j Cypher: MATCH (users:users) WHERE users.name = John RETURN *

                    }

The error response: 

{
  "status":"error_code",
  "message":"error_message/warning_message"
}
Status error response:
  • 0 - success.
  • 199 - warning.
  • 400 - client error.
  • 500 - server error.
