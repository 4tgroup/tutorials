dbConnect Module

To save the database configuration, provide the token to authenticate and create database connection instance for MySQL, SQLite, CouchDB, Neo4j, Redis.

1. Save configuration file

Usage:

var dbConnect = require('./dbAPI-server').dbConnect;
            
var dbinfo = input.metadata.dbinfo;
dbConnect.saveConfig(dbinfo);

Input:

"dbinfo": {
            "DBMS": "mysql/sqlite/couchdb/neo4j/redis",
            "cFile": "",
            "config":{
                "pin": "PIN_CODE",
                "protocol": "http/https/bolt", //bolt is Neo4j protocol
                "host": "DB_HOST",
                "user": "DB_USER",
                "password": "DB_PASSWORD",
                "database": "DB_NAME"
                },
            "connectionString":"" 
        }

Output:

{
  "status":"error_code",
  "message":"error_message/warning_message"
}
Status response:
  • 0 - success. - the configuration file is saved.
  • 400 - client error.
  • 500 - server error.

2. Provide token

Provide the requestID token for dbAPI authentication.

Usage:

var dbConnect = require('./dbAPI-server').dbConnect;
            
var token = dbConnect.saveConfig(payload);

Input:

payload = {
            "user": "USER_NAME",
            "role": "USER_ROLE",
            "secret":"USER_SECRET_INFO",
            "other":""
        }

Output:

{
  "status":"error_code",
  "message":"error_message/warning_message",
  "token":""
}
Status response:
  • 0 - success.
  • 400 - client error.
  • 500 - server error.

3. Create connection instance

            "cFile": "../dbAPI-server/config/dbinfo.neo4j.json",
            "config":{
                "pin": "",
                "protocol": "",
                "host": "",
                "user": "",
                "password": "",
                "database": ""
                },
            "connectionString":"" 
        }

Create connection instance with connectionString (connection string). This is secure connection with the certificate (created by openSSL) and config via Stunnel (See 5. Secure connection configuration).

"dbinfo": {
            "DBMS": "mysql/sqlite/couchdb/neo4j/redis",
            "cFile": "",
            "config":{
                "pin": "",
                "protocol": "",
                "host": "",
                "user": "",
                "password": "",
                "database": ""
                },
            "connectionString":"PROTOCOL:bolt;HOST:127.0.0.1;PORT:7682;cFile:../dbAPI-server/config/dbinfo.neo4j.json" 
        }
cFile in connection string is used to get DB_USER & DB_PASSWORD from configuration file for client authenticates. Connection string supported:
  • CouchDB : "PROTOCOL:http;HOST:127.0.0.1;PORT:59842;DATABASE:db_name;cFile:../dbAPI-server/config/dbinfo.couchdb.json"
  • Neo4j : "PROTOCOL:bolt;HOST:127.0.0.1;PORT:7682;cFile:../dbAPI-server/config/dbinfo.neo4j.json"
  • Redis : "HOST:127.0.0.1;PORT:7682;;DATABASE:0;cFile:../dbAPI-server/config/dbinfo.neo4j.json"
  • MySQL :
    • Native SSL: "USER:sslclient;DATABASE:db_name;CERTFILE:../mysqlCerts/ca-cert.pem"
    • Stunnel: "USER:root;DATABASE:db_name;cFile:../dbAPI-server/config/dbinfo.mysql.json"
  • SQLite: Developing

4. Secret information configuraion

The secret.js secret configuration file located at ../dbAPI-server/config/secret.js. The secret information included:

  • c_passcode : decrypt pin param of dbConnect input
  • secretKey :secret key to create token via JWT
  • tokenAlorithm : alorithm to create token via JWT
  • expireTime : expire time of token created by JWT

5. Secure connection configuration

Config to authenticate with Stunnel SSL for DBMS via openSSL.

Installation:

  • stunnel: https://www.stunnel.org/downloads.html
  • openSSL: https://www.openssl.org/source
  • Cygwin https://www.stunnel.org/downloads.html (for Windows):
  • DBMS: MySQL, Neo4j, Redis, CouchDB. (on Database Server)

Generate self-signed certificatevia openSSL:

  • Generate a key: 
    openssl genrsa -out /etc/stunnel/key.pem 4096
  • Generate the certificate with the expire days define by -days : 
    openssl req -new -x509 -key /etc/stunnel/key.pem -out /etc/stunnel/cert.pem -days 1826
  • Combine the two files: (Use Cygwin for Windows) 
    cat /etc/stunnel/key.pem /etc/stunnel/cert.pem > /etc/stunnel/private.pem
chmod 640 /etc/stunnel/key.pem /etc/stunnel/cert.pem /etc/stunnel/private.pem
    We will combine both the key and certificate into a single file for stunnel to use. We will also change the file permissions to restrict who has access to read these key files. Finally, we got the private.pem certificate.

Config stunnel. Example with Redis (the same configuration for other DBMSs):

  • Config on Database Server (server):
    • Find or create the stunnel.conf configuration file.
    • Add to stunnel.conf file:
      • [redis-server]
pid = /var/run/stunnel.pid //UNIX
accept = 192.168.0.1:6379
connect = 127.0.0.1:6379
cert = private.pem
    • Restart the stunnel

    By default, Redis listen to the localhost, on port 6379. Our configuration has stunnel accept the connections on the external IP (192.168.0.1:6379) and forward the connections to the Redis instance listening on 127.0.0.1:6379.

  • Config on Web Server (client)::
    • Copy the private.pem from Database Server to Web Server.
    • Find or create the stunnel.conf configuration file.
    • Add to stunnel.conf file:
      • [redis-cli]
client = yes
pid = /var/run/stunnel.pid //UNIX
accept = 127.0.0.1:6379
connect = 192.168.0.1:6379
cert = private.pem
    • Restart the stunnel

    On a client instance the accept and connect settings are reversed from the server configuration. This tells stunnel to listen locally on port 6379 (127.0.0.1:6379) and forward the connections to the server host IP with port 6379 (192.168.0.1:6379).
